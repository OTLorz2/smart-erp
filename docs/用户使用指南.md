## 1. 系统简介

### 1.1 什么是Smart-ERP？

Smart-ERP 是一套专门为中小型制造企业（10-30人）设计的企业资源规划管理系统。它可以帮助企业有效管理：

- **库存**：清楚了解仓库里有哪些物料、还剩多少
- **销售**：管理客户订单、发货、收款
- **采购**：管理供应商、进货、付款
- **生产**：安排生产计划、追踪生产进度
- **质量**：把控产品质量、追溯问题源头

### 1.2 系统特点

- **简单易用**：界面简洁直观，操作方便
- **功能全面**：覆盖企业运营的核心环节
- **数据安全**：完善的权限管理，确保数据安全
- **快速部署**：采用现代技术，安装简单

### 1.3 系统架构

Smart-ERP 由两部分组成：

| 组成部分 | 说明 | 访问地址 |
|---------|------|----------|
| 前端界面 | 用户操作界面 | http://localhost:3000 |
| 后端服务 | 数据处理中心 | http://localhost:8000 |

### 1.4 角色说明

系统中的用户分为以下角色：

| 角色 | 权限说明 |
|------|----------|
| **管理员** | 拥有所有权限，可以管理用户、设置系统参数 |
| **业务员** | 负责销售和采购业务 |
| **仓库员** | 负责库存管理、收发货物 |
| **生产员** | 负责生产工单、报工等操作 |
| **质检员** | 负责质量检验、不良品管理 |

## 2. 准备工作

### 2.1 硬件要求

在开始安装之前，请确保您的电脑满足以下要求：

| 项目 | 最低配置 | 推荐配置 |
|------|----------|----------|
| 处理器 | 双核CPU | 四核或以上 |
| 内存 | 4GB | 8GB或以上 |
| 硬盘空间 | 20GB可用空间 | 50GB或以上 |
| 操作系统 | Windows 10/11、macOS、Linux | 同前 |
| 网络 | 稳定的网络连接 | 同前 |

### 2.2 必备软件

在安装Smart-ERP之前，您需要先安装以下软件：

#### 2.2.1 安装Docker Desktop

Docker是运行Smart-ERP的容器平台。请按以下步骤安装：

**Windows系统：**

1. 访问 Docker 官网：https://www.docker.com/products/docker-desktop
2. 点击 "Download for Windows"
3. 下载完成后，双击安装包开始安装
4. 安装过程中会提示需要启用WSL2（Windows Subsystem for Linux），请按照提示操作
5. 安装完成后，重启电脑
6. 启动 Docker Desktop，等待图标显示为绿色

**macOS系统：**

1. 访问 Docker 官网：https://www.docker.com/products/docker-desktop
2. 点击 "Download for Mac"
3. 下载 Intel 芯片或 Apple 芯片（根据您的电脑选择）
4. 双击下载的 .dmg 文件
5. 将 Docker 拖入应用程序文件夹
6. 启动 Docker，等待图标显示为绿色

**验证Docker是否安装成功：**

打开命令提示符（Windows）或终端（macOS/Linux），输入以下命令：

```bash
docker --version
```

如果显示版本号，说明安装成功。

#### 2.2.2 代码编辑器（可选）

建议安装 VS Code 用于查看和编辑项目文件：

1. 访问 https://code.visualstudio.com/
2. 下载并安装适合您系统的版本

### 2.3 获取项目代码

如果您的电脑已经准备好，需要获取Smart-ERP项目代码：

有两种方式获取代码：

**方式一：使用Git克隆（推荐）**

1. 安装 Git：https://git-scm.com/downloads
2. 打开命令提示符或终端
3. 进入您想要存放项目的目录
4. 执行以下命令：

```bash
git clone https://github.com/您的仓库地址/smart-erp.git
```

**方式二：直接下载**

如果您的项目已经在本地，可以直接进入项目目录：

```bash
cd smart-erp
```

## 3. 安装部署

### 3.1 项目目录结构

进入smart-erp目录后，您会看到以下文件和文件夹：

```
smart-erp/
├── backend/          # 后端代码
│   ├── app/         # 后端应用程序
│   └── Dockerfile   # 后端容器配置
├── frontend/        # 前端代码
│   ├── src/        # 前端源代码
│   └── Dockerfile  # 前端容器配置
├── docs/            # 文档
├── docker-compose.yml  # 容器编排文件
└── README.md        # 项目说明
```

### 3.2 启动系统

#### 第一步：确保Docker正在运行

- Windows：检查系统托盘中的 Docker 图标是否显示为绿色
- macOS：检查菜单栏中的 Docker 图标是否显示为绿色

如果Docker未运行，请启动Docker Desktop。

#### 第二步：启动Smart-ERP

1. 打开命令提示符（Windows）或终端（macOS/Linux）
2. 进入smart-erp目录：

```bash
cd smart-erp
```

3. 执行启动命令：

```bash
docker-compose up -d
```

4. 等待服务启动（首次启动需要下载镜像，可能需要几分钟）

5. 验证服务是否启动成功：

```bash
docker-compose ps
```

如果看到以下状态，说明启动成功：

```
NAME                IMAGE               STATUS              PORTS
smart-erp-backend   smart-erp-backend   Up                  0.0.0.0:8000->8000/tcp
smart-erp-frontend  smart-erp-frontend  Up                  0.0.0.0:3000->3000/tcp
```

### 3.3 访问系统

1. 打开浏览器（建议使用 Chrome、Edge 或 Firefox）
2. 在地址栏输入：`http://localhost:3000`
3. 如果看到登录页面，说明系统启动成功

### 3.4 停止系统

当您不需要使用系统时，可以停止服务：

```bash
docker-compose down
```

如果想删除数据库等数据：

```bash
docker-compose down -v
```

### 3.5 重启系统

如果系统出现问题，可以尝试重启：

```bash
docker-compose restart
```

## 4. 首次登录与初始化

### 4.1 默认账户

首次使用时，系统预置了一个管理员账户：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| admin | admin123 | 管理员 |

### 4.2 登录系统

1. 打开浏览器访问 http://localhost:3000
2. 在登录页面输入：
   - 用户名：admin
   - 密码：admin123
3. 点击"登录"按钮

### 4.3 首次使用建议

首次登录后，建议按以下顺序进行初始化：

1. **修改管理员密码**
   - 点击右上角的用户名
   - 选择"个人中心"
   - 修改密码

2. **创建用户账户**
   - 进入"基础数据" -> "用户管理"
   - 点击"新增用户"
   - 为每个员工创建账户

3. **设置基础数据**
   - 仓库信息
   - 物料信息
   - 供应商信息
   - 客户信息

## 6. 基础数据管理

基础数据是整个系统的根基，建议优先设置。

### 6.1 用户管理

管理使用系统的员工账户。

#### 查看用户列表

1. 点击"基础数据" -> "用户管理"
2. 看到所有用户账户列表

#### 新增用户

1. 点击"新增"按钮
2. 填写用户信息：
   - 用户名：登录时使用的名称
   - 姓名：员工真实姓名
   - 角色：选择对应权限角色
   - 密码：登录密码
3. 点击"保存"

#### 编辑/删除用户

- 点击操作列的编辑图标修改用户信息
- 点击删除图标删除用户（需确认）

### 6.2 物料管理

管理企业使用的所有原材料、成品等。

#### 查看物料列表

1. 点击"基础数据" -> "物料管理"
2. 看到所有物料列表

#### 新增物料

1. 点击"新增"按钮
2. 填写物料信息：
   - 物料编码：唯一标识，如 "WL001"
   - 物料名称：如"钢材"、"螺丝"等
   - 规格型号：具体规格
   - 单位：如"个"、"公斤"、"米"
   - 物料分类：原料、半成品、成品
3. 点击"保存"

### 6.3 仓库管理

管理企业的仓库信息。

#### 新增仓库

1. 点击"基础数据" -> "仓库管理"
2. 点击"新增"按钮
3. 填写仓库信息：
   - 仓库编码：如 "CK001"
   - 仓库名称：如"原料仓"、"成品仓"
   - 仓库地址：仓库所在位置
4. 点击"保存"

### 6.4 供应商管理

管理原材料供应商信息。

#### 新增供应商

1. 点击"基础数据" -> "供应商管理"
2. 点击"新增"按钮
3. 填写供应商信息：
   - 供应商编码：如 "GYS001"
   - 供应商名称：公司全称
   - 联系人：负责对接的人员
   - 联系电话：联系方式
   - 地址：公司地址
4. 点击"保存"

### 6.5 客户管理

管理销售客户信息。

#### 新增客户

1. 点击"基础数据" -> "客户管理"
2. 点击"新增"按钮
3. 填写客户信息：
   - 客户编码：如 "KH001"
   - 客户名称：公司全称
   - 联系人：负责对接的人员
   - 联系电话：联系方式
   - 地址：公司地址
4. 点击"保存"

## 7. 库存管理

库存管理是企业运营的核心，确保物料进出库清晰可查。

### 7.1 库存台账

查看当前所有物料的库存情况。

#### 查看库存

1. 点击"库存管理" -> "库存台账"
2. 可以看到：
   - 物料名称
   - 所在仓库
   - 当前库存数量
   - 安全库存（库存低于此数量会预警）

### 7.2 入库管理

物料进入仓库的记录。

#### 采购入库

1. 点击"库存管理" -> "入库单"
2. 点击"新增"按钮
3. 选择入库类型为"采购入库"
4. 填写入库信息：
   - 供应商：选择供货商
   - 仓库：选择入库仓库
   - 入库明细：添加物料及数量
5. 点击"保存"
6. 审核通过后，库存数量会增加

#### 生产入库

生产完成的成品入库，操作类似采购入库，选择类型为"生产入库"。

### 7.3 出库管理

物料离开仓库的记录。

#### 销售出库

1. 点击"库存管理" -> "出库单"
2. 点击"新增"按钮
3. 选择出库类型为"销售出库"
4. 填写出库信息：
   - 客户：选择客户
   - 仓库：选择出库仓库
   - 出库明细：添加物料及数量
5. 点击"保存"
6. 审核通过后，库存数量会减少

#### 生产领料

生产部门领取物料用于生产，选择类型为"生产领料"。

### 7.4 库存盘点

定期核对实际库存与系统库存是否一致。

#### 创建盘点单

1. 点击"库存管理" -> "库存盘点"
2. 点击"新增盘点单"
3. 选择要盘点的仓库
4. 系统会显示当前库存数据
5. 核实实际数量，填写盘点数量
6. 提交盘点结果
7. 根据盘点差异调整库存

### 7.5 库存预警

当库存低于安全库存时，系统会发出预警提醒。

#### 查看预警

1. 点击"库存管理" -> "库存预警"
2. 可以看到所有低于安全库存的物料
3. 及时安排采购，避免缺料

### 8.4 销售退货

3. 填写退货信息：

5. 审核后库存会增加

   - 原入库单：选择对应的入库记录
   - 退货明细：添加退货物料及数量
4. 点击"保存"
5. 审核后库存会减少

## 10. 生产管理

管理生产配方、工艺路线、生产工单等，支持流程制造行业。

### 10.1 配方管理（BOM）

定义产品配方，记录生产一个产品需要哪些物料及用量。

#### 创建配方

1. 点击"生产管理" -> "配方管理"
2. 点击"新增配方"
3. 填写配方信息：
   - 配方编码：唯一标识
   - 配方名称：如"产品A配方"
   - 成品：选择要生产的产品
   - 配方明细：添加所需物料及用量
4. 点击"保存"

**示例**：生产1个"产品A"需要：
- 物料1：2个
- 物料2：5个

### 10.2 工艺路线

定义产品的生产工艺步骤。

#### 创建工艺路线

1. 点击"生产管理" -> "工艺路线"
2. 点击"新增工艺路线"
3. 填写工艺信息：
   - 工艺编码：唯一标识
   - 工艺名称：如"产品A生产工艺"
   - 工艺步骤：添加各道工序
   - 每道工序：工序名称、顺序、预计工时
4. 点击"保存"

### 10.3 生产工单

根据销售订单或库存需求创建生产任务。

#### 创建生产工单

1. 点击"生产管理" -> "生产工单"
2. 点击"新增工单"
3. 填写工单信息：
   - 工单编码：自动生成
   - 产品：选择要生产的产品
   - 计划数量：生产数量
   - 计划开始日期
   - 计划完成日期
   - 工艺路线：选择生产工艺
4. 系统会根据配方自动计算所需物料
5. 点击"保存"

### 10.4 生产报工

每道工序完成后，汇报生产进度。

#### 报工操作

1. 点击"生产管理" -> "生产报工"
2. 选择要报工的生产工单
3. 选择完成的工序
4. 填写：
   - 实际数量：完成数量
   - 作业人员：操作人员
   - 开始/结束时间
5. 点击"提交"

### 10.5 生产进度追踪

查看所有生产工单的执行进度。

#### 查看进度

1. 点击"生产管理" -> "生产进度"
2. 可以看到：
   - 各工单当前状态
   - 已完成工序
   - 正在进行工序
   - 预计完成时间

### 10.6 完工入库

生产完成后，将成品入库。

#### 完工入库操作

1. 点击"生产管理" -> "完工入库"
2. 选择已完成的生产工单
3. 确认入库数量
4. 选择入库仓库
5. 点击"确认入库"
6. 库存数量增加

## 11. 质量管理

把控产品质量，记录检验结果，支持质量追溯。

### 11.1 质检标准

定义各类物料和产品的检验标准。

#### 创建质检标准

1. 点击"质量管理" -> "质检标准"
2. 点击"新增标准"
3. 填写标准信息：
   - 物料：选择物料
   - 检验项目：如外观、尺寸、性能
   - 检验方法
   - 合格标准
   - 检验设备
4. 点击"保存"

### 11.2 来料检验（IQC）

对采购的物料进行检验。

#### 创建来料检验单

1. 点击"质量管理" -> "检验记录"
2. 点击"新增"
3. 选择检验类型为"来料检验"
4. 选择对应的采购入库单
5. 根据质检标准逐项检验：
   - 检验项目
   - 检验结果：合格/不合格
   - 实测值
6. 填写检验结论
7. 点击"保存"

### 11.3 制程检验（IPQC）

生产过程中对半成品的检验。

操作与来料检验类似，选择类型为"制程检验"。

### 11.4 成品检验（OQC）

对生产完成的成品进行检验。

操作与来料检验类似，选择类型为"成品检验"。

### 11.5 不良品管理

记录和处理质量不合格的物料或产品。

#### 创建不良品记录

1. 点击"质量管理" -> "不良品管理"
2. 点击"新增"
3. 填写信息：
   - 不良单号：自动生成
   - 类型：来料不良/制程不良/成品不良
   - 不良物料
   - 数量
   - 不良原因
   - 处理方式：退换货、报废、降级使用
4. 点击"保存"

### 11.6 质量追溯

通过批次号追溯物料或产品的来源和流向。

#### 追溯操作

1. 点击"质量管理" -> "质量追溯"
2. 输入批次号或物料号
3. 查看：
   - 物料来源（采购/生产）
   - 生产过程
   - 检验记录
   - 流向（销售/使用）

## 13. 常见问题与解决方案

### 13.1 启动问题

#### 问题：Docker启动失败

**可能原因：**
1. Docker未安装
2. Docker未启动
3. 端口被占用

**解决方案：**

1. 确认Docker已安装（`docker --version`）
2. 启动Docker Desktop
3. 检查端口8000和3000是否被占用

```bash
# Windows
netstat -ano | findstr "8000"
netstat -ano | findstr "3000"

# macOS/Linux
lsof -i:8000
lsof -i:3000
```

如果端口被占用，可以修改docker-compose.yml中的端口映射。

#### 问题：浏览器无法访问

**检查步骤：**
1. 确认服务是否启动：`docker-compose ps`
2. 确认端口是否正确
3. 尝试使用 http://127.0.0.1:3000 访问
4. 清除浏览器缓存

### 13.2 登录问题

#### 问题：忘记密码

**解决方案：**
1. 联系系统管理员
2. 管理员可在用户管理中重置密码

#### 问题：账号被锁定

**原因：** 连续5次登录失败

**解决方案：**
1. 等待15分钟后自动解锁
2. 联系管理员重置密码

### 13.3 业务操作问题

#### 问题：无法创建订单，提示"库存不足"

**原因：** 库存数量小于订单数量

**解决方案：**
1. 检查库存台账确认实际库存
2. 如需补货，先创建采购单
3. 或者调整订单数量

#### 问题：无法审核单据

**原因：** 没有审核权限

**解决方案：**
1. 确认当前账户角色
2. 联系管理员开启审核权限

### 13.4 数据问题

#### 问题：数据丢失

**可能原因：**
1. 执行了 `docker-compose down -v` 命令
2. 数据库文件损坏

**预防措施：**
1. 定期备份数据
2. 不轻易使用 -v 参数

#### 问题：数据导出为空

**解决方案：**
1. 确认查询条件是否正确
2. 确认日期范围是否有数据
3. 尝试刷新页面后重新查询

### 13.5 性能问题

#### 问题：页面加载慢

**解决方案：**
1. 检查网络连接
2. 清除浏览器缓存
3. 重启服务：`docker-compose restart`

#### 问题：操作无响应

**解决方案：**
1. 刷新页面重试
2. 检查是否有错误提示
3. 重启服务

## 14. 日常维护建议

### 14.1 数据备份

定期备份数据，防止数据丢失。

**手动备份：**

1. 停止服务
2. 复制数据库文件 `smart_erp.db`
3. 存放到安全位置

**备份脚本（可选）：**

```bash
# 创建备份
cp smart_erp.db backup_$(date +%Y%m%d).db
```

### 14.2 日志查看

如遇问题，可以查看系统日志。

```bash
# 查看后端日志
docker-compose logs backend

# 查看前端日志
docker-compose logs frontend

# 实时查看日志
docker-compose logs -f
```

### 14.3 系统更新

1. 拉取最新代码：`git pull`
2. 重新构建镜像：`docker-compose build`
3. 重启服务：`docker-compose up -d`

### 14.4 安全建议

1. **修改默认密码**：首次使用修改admin密码
2. **定期修改密码**：建议每3个月修改一次
3. **限制权限**：按工作需要分配角色权限
4. **注意退出**：离开时点击退出登录

### 14.5 性能优化建议

1. **定期清理不需要的数据**
2. **及时审核和处理单据**
3. **避免高峰期大量查询**
